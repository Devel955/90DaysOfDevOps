# Day 22 – Git Basics Practice

## 🔹 Setup & Configuration

### git --version
Checks installed Git version  
Example:
git --version

### git config --global user.name
Sets global username  
Example:
git config --global user.name "Anand Sen"

### git config --global user.email
Sets global email  
Example:
git config --global user.email "your-email@example.com"

### git config --list
Shows current configuration  
Example:
git config --list


---

## 🔹 Repository Setup

### git init
Initializes a new Git repository  
Example:
git init

### git status
Shows current repository status  
Example:
git status


---

## 🔹 Basic Workflow

### git add
Moves changes to staging area  
Example:
git add git-command.md

### git commit -m "message"
Saves staged changes to repository  
Example:
git commit -m "Add git status command documentation"


---

## 🔹 Viewing History

### git log
Shows detailed commit history  
Example:
git log

### git log --oneline
Shows compact commit history  
Example:
git log --oneline


---

## 🔹 What I Learned Today

- Git tracks changes using commits
- Staging area allows selective commits
- git status is the most important command
- Each commit has a unique ID (SHA)
- Clean commit messages are important
